# 上下文文件选择功能（智能搜索版）

## 功能概述

在AICodeMaster2025插件的聊天界面中，现在新增了"上下文文件选择"功能。用户可以在提问时选择项目内的相关文件，AI将结合这些文件内容来提供更精准、更有针对性的回答。

**最新更新**：现在采用智能搜索下拉框方式，无需打开系统文件选择器，可以快速搜索和选择项目内的文件。

## 功能特点

### 1. 界面位置
- 上下文文件选择面板位于输入框上方
- 采用折叠式设计，带有"上下文文件"标题边框
- 不影响原有的输入框交互功能

### 2. 智能文件选择
- **智能搜索**：点击"选择文件"按钮后，弹出搜索下拉框
- **实时过滤**：在搜索框中输入文件名，实时过滤匹配的文件
- **项目范围**：自动扫描整个项目，只显示文本文件
- **多选支持**：支持同时选择多个文件
- **双击添加**：双击文件可以快速添加
- **相对路径显示**：显示相对于项目根目录的路径，便于识别

### 3. 文件管理
- **查看文件**：选中的文件以列表形式显示，只显示文件名，鼠标悬停可查看完整路径
- **删除文件**：双击列表中的文件项可以删除该文件
- **清空所有**：点击"清空"按钮可以一次性清除所有选择的文件
- **防重复**：自动防止添加重复文件

### 4. 支持的文件类型
自动识别以下文本文件类型：
- **JVM语言**：.java, .kt, .scala, .groovy
- **标记语言**：.xml, .html, .xhtml, .jsp
- **Web技术**：.js, .ts, .json, .css
- **配置文件**：.properties, .yml, .yaml
- **文档**：.md, .txt, .rst
- **其他**：.sql, .gradle, .pom
- **脚本**：.sh, .bat, .cmd
- **其他编程语言**：.py, .rb, .go, .rs

## 使用场景

### 1. 代码分析场景
点击"选择文件"后：
- 在搜索框中输入关键词（如"Service"、"Controller"）
- 选择相关的Java类文件
- 询问：
  - "这几个类之间的关系是什么？"
  - "帮我分析这些类的设计模式"
  - "这些文件中是否存在重复代码？"

### 2. 重构建议场景
通过搜索功能快速定位需要重构的文件：
- 搜索特定模块或包名
- 选择相关文件
- 询问：
  - "请帮我重构这些代码，提高可读性"
  - "这些类的设计有什么问题，如何改进？"
  - "如何优化这些文件的性能？"

### 3. 功能实现场景
利用搜索功能快速找到相关文件：
- 搜索接口名或抽象类
- 选择相关的接口和实现类
- 询问：
  - "根据这些文件，帮我实现一个新的功能"
  - "基于现有代码结构，如何添加新特性？"
  - "请参考这些文件，生成对应的单元测试"

### 4. 配置分析场景
快速搜索项目配置文件：
- 搜索".properties"、".yml"或".xml"
- 选择相关配置文件
- 询问：
  - "请分析这些配置文件的作用"
  - "如何优化这些配置？"
  - "这些配置是否有冲突？"

## 工作原理

### 1. 项目文件扫描
```
初始化 → 扫描项目目录 → 过滤文本文件 → 按文件名排序 → 缓存列表
```

### 2. 智能搜索
```
用户输入 → 实时过滤 → 文件名匹配 → 路径匹配 → 显示结果
```

### 3. 文件选择流程
```
点击选择按钮 → 弹出搜索框 → 输入关键词 → 选择文件 → 添加到列表
```

### 4. 内容处理
```
选择文件 → 读取文件内容 → 缓存到内存 → 构建上下文提示词
```

### 5. 提问处理
```
用户提问 → 检查是否有上下文文件 → 构建包含文件内容的提示词 → 发送给AI
```

## 新增的技术实现

### 1. 智能搜索组件
- 使用`JBPopup`创建可移动、可调整大小的弹出窗口
- `JBTextField`提供实时搜索功能
- `JList`显示搜索结果，支持多选
- 自定义`ListCellRenderer`显示相对路径

### 2. 项目文件管理
- 使用`ProjectFileIndex`扫描项目文件
- 智能文件类型识别
- 实时过滤和搜索
- 防重复添加机制

### 3. 用户交互优化
- **快捷键支持**：实时搜索输入
- **双击操作**：双击文件快速添加
- **多选支持**：按住Ctrl/Cmd键多选文件
- **相对路径显示**：便于理解文件位置

## 提示词格式

当包含上下文文件时，发送给AI的提示词格式如下：

```
以下是相关的上下文文件内容，请结合这些文件内容来回答问题：

### 文件: FileName1.java
```
文件内容...
```

### 文件: FileName2.xml
```
文件内容...
```

---

问题：用户的具体问题

请根据提出的问题作出回答，用中文回答；若需编程，请给出示例，以Java作为默认编程语言输出；如果上下文文件与问题相关，请结合文件内容进行分析和回答。
```

## 注意事项

1. **文件大小限制**：建议选择的文件总大小不要过大，避免超过AI模型的token限制
2. **文件类型**：主要支持文本文件，二进制文件可能显示乱码
3. **内容缓存**：文件内容会在选择时读取并缓存，如果文件在会话期间被修改，需要重新选择
4. **性能考虑**：选择过多或过大的文件可能影响响应速度

## 技术实现

### 1. UI组件
- 使用Swing组件构建界面
- JList显示选择的文件
- JButton处理用户操作
- 集成到现有的CombinedWindowFactory中

### 2. 文件处理
- 使用IntelliJ Platform的FileChooser API
- VirtualFile进行文件读取
- Map缓存文件内容
- 支持文件编码检测

### 3. 状态管理
- 在UIState中维护文件选择状态
- 支持多项目独立的文件选择
- 自动清理资源

这个功能大大增强了AI助手的上下文理解能力，让开发者能够获得更精准、更有针对性的技术支持。
